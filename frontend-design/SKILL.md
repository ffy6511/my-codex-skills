This skill guides creation of distinctive, production-grade frontend interfaces that avoid generic "AI slop" aesthetics. Implement real working code with exceptional attention to aesthetic details and creative choices.

The user provides frontend requirements: a component, page, application, or interface to build. They may include context about the purpose, audience, or technical constraints.

## Design Thinking

Before coding, understand the context and commit to a bold aesthetic direction:
- **Purpose**: Identify what problem the interface solves and who uses it.
- **Tone**: Pick a clear point of view: brutally minimal, maximalist chaos, retro-futuristic, organic, luxury, playful, editorial, brutalist, art deco, pastel, industrial, or another strong direction that fits the request.
- **Constraints**: Respect framework, performance, accessibility, and product constraints.
- **Differentiation**: Decide what makes the experience memorable.

Choose a clear conceptual direction and execute it with precision. Bold maximalism and refined minimalism both work; intentionality matters more than intensity.

Then implement working code that is:
- Production-grade and functional
- Visually striking and memorable
- Cohesive, with a clear aesthetic point of view
- Meticulously refined in spacing, typography, states, and motion

## Frontend Aesthetics Guidelines

Focus on:
- **Typography**: Choose fonts with character. Avoid generic defaults such as Arial, Inter, Roboto, and system stacks unless the existing product already depends on them. Pair a distinctive display font with a refined body font when appropriate.
- **Color and Theme**: Commit to a cohesive palette. Use CSS variables for consistency. Strong dominant colors with deliberate accents usually outperform timid, evenly distributed palettes.
- **Motion**: Use animation intentionally. Prefer CSS-first motion for simple interfaces, and use a motion library when the stack already includes one. Prioritize a few high-impact moments such as page-load reveals, hover transitions, or scroll-based emphasis.
- **Spatial Composition**: Use asymmetry, overlap, grid-breaking elements, diagonal flow, negative space, or controlled density when they support the concept.
- **Backgrounds and Visual Details**: Build atmosphere instead of defaulting to flat fills. Use gradient meshes, noise, patterns, layered transparencies, shadows, borders, grain, or custom cursors when they fit the design language.

Never fall back to generic AI-generated aesthetics such as overused font families, purple-on-white gradients, predictable layouts, or cookie-cutter component patterns. Make unexpected choices that feel designed for the actual context.

## Implementation Rules

- Match implementation complexity to the aesthetic vision. Maximalist designs can justify elaborate effects; minimalist designs require restraint and precision.
- Preserve the existing visual language when working inside an established product or design system, unless the user asks for a fresh direction.
- Ensure the result works on both desktop and mobile.
- Keep the interface usable and accessible while pursuing visual distinction.
- Prefer polished, working code over moodboard-style mockups.

Remember: do not hold back creatively. Commit fully to a distinctive vision and execute it with care.
