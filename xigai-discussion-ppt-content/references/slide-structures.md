# Slide Structures

Use this file to choose a structure based on the actual topic.
Do not apply one fixed pattern to all themes.

## 1) Concept Interpretation Topic

Use when the topic asks "how to understand X" or "what is X".

Suggested flow:
1. Define the concept and boundaries.
2. Explain why this concept emerges from China's conditions.
3. Show practical value with current achievements/cases.
4. Conclude with implications for present development.

## 2) Historical Evolution Topic

Use when the topic focuses on "how it formed" or "historical process".

Suggested flow:
1. Starting conditions/background.
2. Key stage transitions and milestones.
3. Mechanisms that drove success.
4. Lessons and contemporary meaning.

## 3) Path Justification Topic

Use when the topic asks "why this path/model is necessary".

Suggested flow:
1. Clarify the evaluation criteria.
2. Compare feasible paths at principle level (avoid simplistic binary rhetoric).
3. Provide evidence from governance, economy, social outcomes, or modernization progress.
4. Summarize why the selected path is historically and practically grounded.

## 4) Policy Effectiveness Topic

Use when the topic asks about policy outcomes.

Suggested flow:
1. Policy objective and problem setting.
2. Implementation mechanism.
3. Observable outcomes and data.
4. Remaining issues and optimization direction.

## 5) Comparative Analysis Topic

Use when the topic asks for comparison across ideas, periods, or systems.

Suggested flow:
1. Define dimensions and scope of comparison.
2. Compare under each dimension with fair criteria.
3. Explain root causes of differences.
4. Return to the class topic with a balanced conclusion.

## Practical Notes

- Keep one "main question" through all slides.
- Keep each slide to one core message.
- Prefer "claim -> evidence -> implication" inside each slide.
